# 🦝 [小浣熊ERP官网](https://samforo.icu/) - 物流中心 使用教程

## 使用步骤

1. 📥 下载模板: [物流中心导入模板.xlsx](https://github.com/68110923/chrome_plugins/raw/main/docs/image/LOGISTICS_CENTER/物流中心导入模板.xlsx) *(右键另存为下载)*
2. ✏️ 编辑模板表格字段
   - **订单号**: 唯一字段,每个订单号只能生成一个物流单号 (会在物流面单上显示, 如果遇到一个订单号需要多个物流单号的情况下, 可以在订单号后添加 1,2,3 等序号防止冲突)
   - **收件人信息**: 收件人姓名,手机号,地址,邮编 等等  (会在物流面单上显示)
   - **预计到货日期**: 格式为 2023-12-31 (添加轨迹的时候会根据这个日期来生成预计到达的时间, 需要大于当前日期)
3. 📤 上传表格到: 小浣熊ERP - `物流中心` - `待发货物流`
4. 🛍️ 购买物流号: `待发货物流` 内勾选需要物流号的订单 → 点击 **购买JC物流号:无轨迹**
5. ⚙️ 生成物流轨迹: 购买之后会订单会流转到`无轨迹物流`, 这个时候物流单号已经生成了, 相当于有了物流单号但是未被快递员揽收的状态
6. ➕ 添加轨迹: `无轨迹物流` 勾选需要添加轨迹的订单 → 点击 **添加轨迹** → 选择需要的轨迹类型 → 提交即可

## 轨迹说明

### 特殊轨迹: 已揽收
- **状态说明**: 表示快递员已经揽收了订单, 但是没有后续轨迹
- **流转规则**: 订单状态会变成 **已揽收**, 自动流转到 `已揽收物流`
- **后续操作**: 后续如果有其他轨迹, 可以在 `物流中心` - `已揽收物流` 内添加后续轨迹

### 正常签收场景: 正常轨迹
- **状态说明**: 表示快递员正常揽收了订单, 并且补全了物流轨迹
- **流转规则**: 每到一个时间节点就会自动添加系统预设的轨迹, 自动流转到 `已发货物流`
- **后续操作**: `已发货物流` 内的订单如果业务上处理完成了,可以 勾选 → **标记为已完成**, 订单会自动流转到 `已完成物流`

### 异常签收场景: 妥投失败\收件人拒收\收件人联系不上\无有效地址\货物已退回给发件人
- **状态说明**: 表示快递员正常揽收了订单, 并且补全了物流轨迹, 但是最后一步本应该签收的轨迹改为异常场景
- **流转规则**: 每到一个时间节点就会自动添加系统预设的轨迹, 自动流转到 `已发货物流`
- **后续操作**: `已发货物流` 内的订单如果业务上处理完成了,可以 勾选 → **标记为已完成**, 订单会自动流转到 `已完成物流`
